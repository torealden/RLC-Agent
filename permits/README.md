# Organized Permits Archive

Per-facility folders containing source PDFs, equipment lists (CSV),
extraction summaries, and process-flow coverage spot-check reports.

Generated by `scripts/publish_permits_to_organized_archive.py`. Reads
from `bronze.state_air_permits` + `bronze.state_air_permit_units` and
copies source PDFs from `collectors/epa_echo/raw/`.

## Structure

```
permits/
├── <industry>/                 (oilseed_crush, biodiesel, etc.)
│   └── <state>/                (ia, in, ...)
│       └── <facility_slug>/
│           ├── source.pdf                  Original Title V PDF
│           ├── equipment_list.csv          Excel-friendly EU list
│           ├── extraction_summary.md       Narrative summary
│           └── process_flow_coverage.md    Canonical-flow spot-check
└── README.md
```

## Spot-checking workflow

1. Open `process_flow_coverage.md` for the facility — it shows which
   canonical oilseed-crush steps have extracted equipment vs which are
   missing (⚠️). Missing steps could be: (a) LLM missed it, (b) permit
   doesn't itemize it, or (c) the facility doesn't have that capability.
2. Open `source.pdf` and check pages 4-5 (Equipment List Table in the
   IDNR format). Cross-reference against `equipment_list.csv`.
3. Discrepancies inform W2 hybrid-extraction work (regex on the table +
   LLM on narrative).

## Re-publishing

Run after any new permit extraction or bronze update:
```
python -m scripts.publish_permits_to_organized_archive
```
Idempotent — overwrites existing files in place.
